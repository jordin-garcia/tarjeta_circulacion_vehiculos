 Proyecto final : 
El presente proyect o tiene como objetivo aplicar de manera progresiva el 
conocimiento adquirido en el curso . 
 
Parte I, a entregar el día 25 de marz o de 2026 :  
Instrucciones:  Debe realizar  el análisis y levantamiento de información contenida 
en la tarjeta de circulación vehicular guatemalteca, identificando datos relevantes y 
la estructura que sigue para que se pueda proveer el servicio a todos los vehículos 
que circulan a diario en Guatema la, y los vehículos de nuevo ingreso.  
Resultados esperados:  
• Modelo entidad -relación.  
• Modelo relacional  
• Entidades identificadas.  
• Lista de atributos identificados para cada entidad.  
• Comprensión del dominio de cada atributo.  
• Identificación de redundancia.  
• Claves primarias y foráneas  
• Implementación en motor PostgreSQL.  
 
Parte II, a entregar el día 13 de mayo de 2026:   
Instrucciones:  Debe realizar  una aplicación cliente -servidor base, donde la 
aplicación “cliente” permite consultar información de la tarjeta de circulación y el 
servidor accede a la base de datos PostgreSQL. Esta fase busca demostrar la 
integración entre la base de datos y una aplica ción real.  
Resultados esperados:  
• Aplicación funcional  
• Conexión  cliente -servidor  
• Modelado de base de datos corregido
